一. 使用框架进行开发

1.什么是框架？

一个精装房，不需要你装修，你只需要把你想用的东西搬到房子里摆好即可入住。

2.框架开发的好处

减少底层代码的反复书写，只需调用框架的方法就能实现你想要的功能。能够极大的提高项目的编写效率。

二. 如何在项目中使用框架

1.使用vue-cli脚手架前的准备工作

  1. 安装和配置nodejs
  2. 使用 npm全局安装 webpack
  3. 全局安装 vue-cli

2.使用vue-cli快速创建项目

  1. 找个文件夹，文件夹下打开终端
  2. vue create xxx创建项目
  3. cd xxx进入项目目录
  4. npm install
  5. npm run serve

三. 使用vue脚手架创建的项目结构

  • public :存放网站页签图标和heml文件
  • src/assets ：存放静态资源
  • src/components ：存放所有组件
  • src/App.vue :是存放所有组件的父组件
  • src/main.js :执行npm run serve 后首先运行的文件

四.Vm实例对象中的属性

  • el: 指定 vm 的挂载元素，也可通过 $mount(“#root”) 指定挂载点

  • data: 当一个 Vue 实例被创建时，它向 Vue 的响应式系统中加入了其 data 对象中能找到的所有的属性。当这些属性的值发生改变时，html 视图将也会产生相应的变化。

  • methods: 用来存放触发事件的内容

  • computed: 计算属性，computed 是通过 data 中的已有属性拼凑而成。借助了Object. defineProperty 的 getter 和 setter。 get 初始化和每次 computed 依赖 data 中的数据改变时调用。与 methods 相比，有缓存机制。计算属性最终会出现在 vm 上，计算属性被修改时，必须写 set 函数改变 data 中的依赖数据，进而引起计算属性的变化。

  • watch: 监听属性，当监视的属性变化，handler 自动调用。通过 watch 配置或 $watch 监视

五.V-指令

1.数据绑定：v-bind v-model

  • Vue中有2种数据绑定的方式：
    1. 单向绑定(v-bind)：数据只能从data流向页面。
    2. 双向绑定(v-model)：数据不仅能从data流向页面，还可以从页面流向data。
  • 备注：
    1. 双向绑定一般都应用在表单类元素上（如：input、select等）
    2. v-model:value 可以简写为 v-model，因为v-model默认收集的就是value值。

2.条件渲染：v-if v-else-if v-else v-show

    1. v-show 展示或者不展示元素
    1. v-if 创建或者销毁元素
      备注：
      • 非常频繁地切换，则使用 v-show 较好
    • 如果在运行时条件很少改变，则使用 v-if 较好

3.渲染文本: v-text v-html

  • v-text：向其所在的节点中渲染文本内容（innerText）
  • v-html：向指定节点中渲染包含html结构的内容（ innerHTML）
    与插值语法的区别：
    1. 会替换掉节点中所有的内容，则不会
      严重注意：v-html有安全性问题！！！！
      1. 在网站上动态渲染任意HTML是非常危险的，容易导致XSS攻击。
      2. 一定要在可信的内容上使用v-html，永不要用在用户提交的内容上！

4.循环渲染: v-for

  • 当我们有一组数据需要进行渲染时，我们就可以使用v-for来完成。
    1. v-for的语法类似于JavaScript中的for循环
    2. 遍历数组或者对象，配合key属性使用
